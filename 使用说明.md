# 使用说明

Memory Framework 是一个 AI 辅助小说写作框架，帮助你管理角色状态并生成连贯的章节内容。

## 一、安装配置

### 1.1 安装依赖

```bash
cd memory_framework
pip install -r requirements.txt
```

### 1.2 配置 API

复制配置模板：

```bash
cp config.example.json data/config.json
```

编辑 `data/config.json`，填入你的 API 信息：

```json
{
  "api_url": "https://api.siliconflow.cn/v1",
  "api_key": "sk-your-api-key-here",
  "model": "deepseek-ai/DeepSeek-V3"
}
```

支持任何兼容 OpenAI 格式的 API。

### 1.3 启动服务

```bash
python agent_web.py
```

浏览器访问：http://localhost:5001

## 二、使用流程

### 2.1 创建项目

1. 点击"创建新项目"
2. 填写项目信息：
   - 项目名称
   - 小说标题
   - 大纲描述
   - 预计章节数

### 2.2 添加角色

1. 进入"角色管理"页面
2. 点击"添加角色"
3. 填写角色信息：
   - 角色名称
   - 初始状态（位置、情绪、目标等）

### 2.3 配置字段模式

在"模式配置"页面可以：
- 添加新的状态字段
- 设置必填/可选字段
- 定义更新规则

### 2.4 生成章节

1. 进入"章节生成"页面
2. 选择要生成的章节
3. 点击"开始生成"
4. 实时查看生成进度：
   - 状态读取
   - 内容生成
   - 状态更新

### 2.5 查看和编辑

生成完成后可以：
- 查看章节内容
- 查看角色状态变化
- 手动编辑章节
- 查看章节摘要

## 三、核心概念

### 3.1 双智能体架构

**StateAgent（状态智能体）**
- 负责读取和更新角色状态
- 使用工具调用管理状态
- 记录状态变化原因

**ContentAgent（内容智能体）**
- 专注于生成章节文本
- 不直接操作状态
- 接收清理后的上下文

### 3.2 仅追加历史

角色状态不会被覆盖，而是追加新记录：

```json
{
  "fields": {
    "location": [
      {
        "value": "华山派",
        "chapter": 1,
        "reason": "故事开始地点",
        "timestamp": "2024-03-09T10:00:00"
      },
      {
        "value": "洛阳城",
        "chapter": 5,
        "reason": "追查线索",
        "timestamp": "2024-03-09T11:30:00"
      }
    ]
  }
}
```

这样可以：
- 查看角色状态演变
- 回溯到任意章节的状态
- 理解状态变化原因

### 3.3 上下文构建

生成章节时，系统会自动组装：
- 项目大纲
- 最近章节摘要
- 相关角色当前状态
- 章节生成提示

## 四、高级功能

### 4.1 多项目管理

支持同时管理多个小说项目，每个项目独立：
- 独立的角色库
- 独立的章节内容
- 独立的状态模式

### 4.2 状态模式定制

可以根据小说类型定制状态字段：

**武侠小说**：武功、内力、门派、江湖地位
**科幻小说**：科技等级、星球位置、资源
**现代都市**：职业、财富、人际关系

### 4.3 实时进度查看

生成过程中可以看到：
- 当前执行的步骤
- 工具调用详情
- 状态更新记录
- 生成的内容片段

## 五、常见问题

### 5.1 API 调用失败

**问题**：提示 API 错误

**解决**：
1. 检查 `data/config.json` 中的 API 配置
2. 确认 API Key 有效
3. 确认 API URL 正确
4. 检查网络连接

### 5.2 生成内容不连贯

**问题**：章节内容与前文不符

**解决**：
1. 检查角色状态是否正确更新
2. 查看章节摘要是否准确
3. 适当增加上下文窗口
4. 手动编辑不准确的状态

### 5.3 状态更新失败

**问题**：角色状态没有更新

**解决**：
1. 查看生成日志中的工具调用
2. 检查状态模式配置
3. 确认字段名称正确
4. 手动补充缺失的状态

### 5.4 生成速度慢

**问题**：生成一章需要很长时间

**解决**：
1. 使用更快的模型
2. 减少上下文长度
3. 简化状态字段
4. 检查 API 限流设置

## 六、最佳实践

### 6.1 角色设计

- 初始状态尽量详细
- 关键字段设为必填
- 定期检查状态一致性

### 6.2 大纲规划

- 明确每章的核心事件
- 标注关键转折点
- 预留伏笔和呼应

### 6.3 状态管理

- 重要变化及时更新
- 记录清晰的变化原因
- 定期回顾状态演变

### 6.4 内容生成

- 先生成少量章节测试
- 根据效果调整提示词
- 必要时手动干预编辑
